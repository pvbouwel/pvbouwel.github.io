Podman pitfalls

November 25, 2023 One-minute read
Linux • Containers

Raw notes Link to heading

This is not a full post but rather some common issues encountered with Podman and pointers to solutions.

Fedora docs are your friend Link to heading

https://docs.fedoraproject.org/en-US/fedora-coreos/

SELinux might be on Link to heading

If you are having permission denied errors watch out for SELinux. Check your podman VM and verify /etc/selinux/config. You can consider switching to permissive mode + reboot

Certificate errors Link to heading

MITM Link to heading

Some company like to or must inspect their users traffic. Generally this is done by having a transparent proxy which terminates SSL/TLS and uses a self-signed certificate that is owned by the company and can be considered trusted. The default podman VM won´t trust this certificate. You can try the following:

COPY the PEM file to /etc/pki/ca-trust/source/anchors/ and then update the trust: update-ca-trust force-enable && update-ca-trust extract

Time drift Link to heading

If the podman VM has time drift this can also break SSL/TLS certificate verification. Just update the time of your VM.

Allow docker in podman Link to heading

Docker in Docker (dind) or Docker in podman does not always work in the Fedora VM you should install podman-docker package

sudo rpm-ostree install podman-docker
